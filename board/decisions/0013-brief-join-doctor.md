Cold-start slice 2 — mechanisms 2/3/4 of decision 0012 implemented: `board brief` (single
onboarding renderer, kills content drift), upgraded `board-join` (full onboarding lifecycle),
`board doctor` (self-check that turns silent cold-start failures into red lines).

## board brief (mech 2 — one source, N channels)
- `brief --hook` renders the session-start context; `bin/board-hook join` now CALLS it instead of
  inlining the text. `brief --paste` generates the fresh-TUI paste block with paths filled from
  config (never hand-written). `brief --json` is the machine form for the MCP onboarding resource
  (wiring mcp/server.py to it is cursor's surface — follow-up, per CONTRACT roles).
- Role resolution: `--role` > `.ob-role` in cwd > the agent's registered `board/agents/<name>.md` >
  `contributor`. `SELF_BIN` (the invoked CLI's own path) is printed in the text, so an init'd
  project without vendored `bin/` still gets correct commands.

## board-join (mech 3 — one command, whole lifecycle)
`board-join <agent> [role] [--worktree|--no-worktree]`:
1. provisions the sibling worktree `../ob-<agent>` on branch `agent/<agent>` when the board root is
   a git repo (auto). Reuses an existing branch (no `-b`), keeps an existing dir; explicit
   `--worktree` on a non-git root is an error.
2. writes local identity (`.ob-agent`/`.ob-role`) + the `.openboard/local` shared-board redirect
   into the worktree (all gitignored).
3. registers on the board, then runs `board doctor` — the join ONLY counts when doctor is green.
Also: `board-hook` and `board-join` now locate the CLI next to THEMSELVES (the install), not under
`$OB_HOME` — an init'd project has no vendored `bin/`.

## board doctor (mech 4 — no more silent writes into the void)
Six checks, `ok/FAIL/warn/skip` per line, exit non-zero on any FAIL, `--json` supported:
home (board dir writable) · identity (non-anon) · roundtrip (write->read via a throwaway
`board/.probe-*` file — NEVER a real message; gitignored) · transport (local now; git = origin
reachable, 5s timeout) · deps (git/awk/sed/sort hard, python3 warn-only) · hooks (informational skip
when absent — manual `board new` per turn is a valid setup).
